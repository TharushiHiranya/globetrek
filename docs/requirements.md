## Scenario summary

GlobeTrek Adventures is a newly established travel and tourism company in Negombo. The brief asks for a web-based platform where:
- Customers can browse packages, book trips, and submit queries.
- Staff can manage bookings, update packages, and respond to customer inquiries.
- Admins can manage staff accounts, oversee bookings, generate reports, and handle data security.

## Tasks and marks

**Task 01: Plan and design (40 marks)**
- 01a: Compare at least two similar travel websites (20 marks).
- 01b: UI designs and a site map, with justifications for design decisions (20 marks).

**Task 02: Implementation (40 marks)**
- 02a: Frontend implementation (20 marks).
- 02b: Backend implementation (20 marks).

**Task 03: Testing (20 marks)**
- Testing overview, test plan, test cases with results, user feedback, and a user manual.

Total word count is 3000 words.

## Grading notes

The pass mark is 40% for undergraduate work.

To reach the Good band (55-69%):
- Compare at least two similar sites.
- Include wireframes covering most of the system.
- Build an attractive, easy-to-navigate frontend.
- Provide a fully functional, error-free system with proper database transactions.

To reach Excellent (70-100%):
- Provide in-depth competitor comparison with critical evaluation.
- Show detailed wireframes with color suggestions.
- Show eye-catching design with strategic use of white space.
- Give a well-prepared demonstration and explain code.
- Present strong acceptance testing evidence.
